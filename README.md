# Burnout AI Risk Prediction

An end-to-end Machine Learning project focused on predicting student burnout risk levels based on academic performance, AI usage patterns, study habits, and behavioral indicators.

The project is designed using modular software engineering principles with a strong focus on scalability, maintainability, reproducibility, and future MLOps integration through tools such as DVC, MLflow, and automated training pipelines.

# Directory Overview

## artifacts/

Stores all pipeline-generated outputs.

Examples include:

* Component artifacts
* Metadata files
* Validation reports
* Model artifacts
* Intermediate pipeline outputs

This directory acts as the communication layer between pipeline stages and can later be tracked using DVC.

---

## configs/

Contains project-level configuration files.

Examples:

* Dataset schema definitions
* Pipeline configurations
* Model parameters
* Environment-specific settings

Keeping configurations separate from source code allows easier maintenance and experimentation.

---

## data/

Central location for all project datasets.

Common lifecycle:

```text
Raw Data
    ↓
Interim Data
    ↓
Processed Data
```

Subdirectories help maintain clear separation between original datasets and transformed versions.

---

## docs/

Project documentation and static documentation assets.

Can be used with documentation frameworks such as MkDocs to generate browsable project documentation.

# Design Principles

The project follows several software engineering principles commonly used in production-grade ML systems:

* Single Responsibility Principle (SRP)
* Dependency Injection
* Loose Coupling
* Configuration-Driven Development
* Artifact-Based Communication
* Modular Pipeline Architecture
* Reproducible Experimentation
* Future DVC & MLflow Compatibility

# High-Level Pipeline

```text
Data Ingestion
        ↓
Data Validation
        ↓
Data Transformation
        ↓
Model Training
        ↓
Model Evaluation
        ↓
Prediction
```

Each stage produces artifacts that become inputs for downstream stages, creating a reproducible and scalable machine learning workflow.
